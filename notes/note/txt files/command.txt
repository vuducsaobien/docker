- 1. Xoá tất cả container (Nếu không có container nào → nó không chạy lệnh → không báo lỗi.)
    docker ps -aq | xargs -r docker rm -f

- 2. Xoá tất cả images
    docker images -aq | xargs -r docker rmi -f

- 3. Xoá tất cả volumes
    docker volume ls -q | xargs -r docker volume rm -f

- 4. Xoá tất cả networks (trừ mặc định)
    docker network rm $(docker network ls | grep -v "bridge\|host\|none" | awk '{print $1}')
    docker network ls | grep -v "bridge\|host\|none" | awk '{print $1}' | xargs -r docker network rm

- 5. Xoá toàn bộ Build cache (BuildKit)
    Không tìm được command để xoá tất cả build trong Build > Build History
    => Phải xoá tất cả các Build = GUI Docker Desktop

- 6. Liệt kê tất cả mạng Docker.
    - docker network ls

        5fe5907c34ed   bridge    bridge    local
        0efb45f24983   host      host      local
        d99401f9cabb   none      null      local

- 7. Thống kê dung lượng chi tiết (images, containers, volumes, build cache).
    docker system df -v

        Containers space usage:

        CACHE ID   CACHE TYPE   SIZE      CREATED   LAST USED   USAGE     SHARED

- 8. Pull 1 image về 
    docker pull docker/welcome-to-docker

- 9. Tạo container từ image:
    - docker run [OPTIONS] <image-name>
    - docker run -d -p 8080:80 docker/welcome-to-docker
        1. Tải image docker/welcome-to-docker từ Docker Hub (nếu chưa có sẵn).
        2. Tạo một container mới từ image đó.
        3. Chạy web server nhỏ bên trong container (thường là Nginx hoặc app Node.js demo).
        4. Container này lắng nghe port 80 bên trong, được bạn map ra 8080 bên ngoài.

- 10. Tạo và chạy một container mới từ image đã chỉ định.
    docker run <the-container-id-or-container-name>

- 11. Dừng container (nếu đang chạy)
    docker stop <the-container-id-or-container-name>

- 12. Chạy lại (start lại) container cũ
    docker start (sử dụng lại container cũ, giữ nguyên volume, network, log, v.v.)

- 13. Nếu muốn “restart” trong một lệnh duy nhất:
    docker restart (= `docker stop` and `docker start`)

- 14. docker run --name nginx_container nginx (Chạy 1 container name = nginx_container từ image nginx)

- 15. Xem log đầu ra (stdout / stderr) của 1 container
    docker logs -n 5 nginx_container (Xem log 5 dòng cuối cùng container name = nginx_container)
    docker logs nginx_container (hết log luôn)
    
- 16. docker attach nginx_container (Phải start container trước)
docker ps -a

- 17. Để biết 1 container thuộc về network nào ?
    - Cách 1: docker inspect <container_name_or_id> => 
        `docker inspect <container_name_or_id>`
        docker inspect container_nginx
            => "Networks": {
                    "bridge": { (container này thuộc network Bridge)
                    ....
                    }
                }

        Cách ngắn hơn
            `docker inspect -f '{{json .NetworkSettings.Networks}}' container_nginx | jq`

    Cách 2: Xem tất cả các Container trong 1 Network:
        - Liệt kê tất cả các Network hiện có:
            + docker network ls
                NETWORK ID     NAME      DRIVER    SCOPE
                f31f37cdd793   bridge    bridge    local
                0efb45f24983   host      host      local
                d99401f9cabb   none      null      local

        - Xem tất cả các Container trong 1 Network:
            + docker network inspect bridge
                ................
                "Containers": {
                    "fbe80fe6ce08e292041637b966e7e5b7ac2f5b9a0138e735e01ea1d70c7c7d0f": {
                        "Name": "container_nginx",
                        "EndpointID": "e7124d4a7e9abe7f8b8b5eadcc01c74222ef24ef823153a7b9d507482cc5e6de",
                        "MacAddress": "aa:79:4d:c2:7a:14",
                        "IPv4Address": "172.17.0.2/16",
                        "IPv6Address": ""
                    }
                },
                ................


    => container container_nginx, đang nằm trong bridge network,
    và bạn muốn chuyển sang network my_network mà không xoá container, không chạy lại.

    Bước 1: 
        - Kiểm tra Status Container: docker ps

            CONTAINER ID   IMAGE     COMMAND                  CREATED          STATUS          PORTS     NAMES
            fbe80fe6ce08   nginx     "/docker-entrypoint.…"   56 minutes ago   Up 56 minutes   80/tcp    container_nginx
            => container_nginx đang chạy

        - Kiểm tra current network của Container này:
            `docker inspect -f '{{json .NetworkSettings.Networks}}' container_nginx | jq`
                "bridge": { .....
                    "IPAMConfig": null,
                    "Links": nu
                } .....
            => Container đang thuộc Network bridge

    Bước 2: Tạo 1 Custom Network riêng
        - docker network create my_network
        Kiểm tra: `docker network ls`
            NETWORK ID     NAME         DRIVER    SCOPE
            f31f37cdd793   bridge       bridge    local
            0efb45f24983   host         host      local
            517d337c51ae   my_network   bridge    local
            d99401f9cabb   none         null      local

    Bước 2: Ngắt container khỏi network hiện tại
        docker network disconnect bridge container_nginx
        Kiểm tra current network của Container này:
            {}
            => Container không có Network nào

    Bước 3: Kết nối container sang network khác
        docker network connect my_network container_nginx
        Kiểm tra lại: 
            `docker inspect -f '{{json .NetworkSettings.Networks}}' container_nginx | jq`
            {
                "my_network": {
                    "IPAMConfig": {},
                    .....
                }
            }
            => Đã chuyển Network của container từ bridge sang my_network
